# ASSISTANT_DM

Веб‑чат‑виджет и бэкенд для **консультаций по корпоративным тренингам бизнес‑тренера Дмитрия Мальцева**: виртуальный ассистент «Александр» отвечает на вопросы о программах, ценах и форматах, мягко ведёт диалог и **собирает заявки** на звонок специалиста.

## Зачем проект

- **Посетителю сайта** — быстрые ответы по тренингам в режиме диалога без звонка менеджеру.
- **Заказчику / тренеру** — единый канал сбора лидов: заявки с именем, телефоном и удобным временем связи попадают в БД и уходят уведомлением в **Telegram** (опционально дубль на email).

## Что делает система

1. **Чат** — клиент (страница `/ui`) отправляет сообщения на `POST /api/chat`; сервер держит сессию (`session_id`), хранит историю в SQLite и вызывает LLM с системным промптом и блоком **«База знаний»** из текстовых файлов `data/*.txt`.
2. **Ответы** — генерируются через OpenAI‑совместимый API (**ProxyAPI**); поведение задаётся `prompt_system.txt` (роль, тон, правила про цены и заявки).
3. **Заявка (лид)** — модель может вызвать инструмент `submit_lead`; бэкенд валидирует данные, сохраняет запись в таблицу `leads`, отправляет текст в Telegram и возвращает пользователю подтверждение.
4. **Здоровье** — `GET /health` для проверки, что приложение поднято.

Подробности по интеграциям, схеме лида и рекомендациям см. в [`DEVELOPER_SPEC.md`](DEVELOPER_SPEC.md).

## На чём реализовано

| Слой | Технологии |
|------|------------|
| Язык | **Python 3** |
| HTTP API | **FastAPI**, **Uvicorn** |
| Данные | **SQLAlchemy 2**, **SQLite** (путь задаётся в `DATABASE_URL`; схема — [`schema.sql`](schema.sql)) |
| Конфиг | **pydantic-settings**, **python-dotenv** (`load_env()` в точке входа) |
| LLM | SDK **OpenAI** к совместимому эндпоинту (**ProxyAPI** по умолчанию) |
| Статика | HTML/CSS/JS в каталоге **`static/`** — виджет чата |

## Быстрый старт

1. Создайте виртуальное окружение и установите зависимости:

   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. Скопируйте переменные из **[`EnvExample`](EnvExample)** в файл **`.env`** в корне проекта и заполните как минимум **ключ API** (`OPENAI_API_KEY` или `PROXY_API_KEY`) и при необходимости **Telegram** для лидов (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_LEAD_CHAT_ID`).

3. Запуск сервера:

   ```bash
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

4. В браузере: **http://127.0.0.1:8000/ui** — виджет чата; API — **http://127.0.0.1:8000/api/chat** (тело JSON: `session_id`, `message`).

Проверка Telegram (после настройки `.env`): `python scripts/test_telegram.py`.

## Важные файлы

| Путь | Назначение |
|------|------------|
| `app/main.py` | Точка входа FastAPI, маршруты, раздача `/ui` |
| `app/chat.py` | Диалог, история, вызов LLM, обработка `submit_lead` |
| `app/lead_service.py`, `app/telegram_notify.py` | Лиды и уведомления |
| `app/knowledge.py` | Загрузка и сборка блока знаний из `data/*.txt` |
| `prompt_system.txt` | Системный промпт ассистента |
| `data/*.txt` | Материалы по тренингам, ценам, контактам (не подставляйте секреты в репозиторий без необходимости) |
